Comportement

Cette partie concerne la définition des classes permettant de représenter une caisse et son contenu (argent) ainsi que l'objet qui représente le change (un ensemble de pièces billets correspondant à un montant).

  • Le scan d'un code barre via l'utilisation d'un Intent (qui démarre la caméra de la tablette) provoque :
    • 1 point code l'ajout à la liste du produit s'il existe dans l'inventaire.
    • 1 point code l'ouverture du dialogue d'ajout à l'inventaire si le code barre n'existe pas. TODO SCREENSHOT
  • 1 point code Le paiement provoque :
    • la sauvagarde de l'achat de cette liste
    • l'affichage de tous les achats (montant + date + liste des produits avec quantité) depuis l'installation de l'application dans le log de la tablette avec le tag "ACHATS"
    • l'effacement de la liste d'achats courante.

Tiroir caisse et change calculent le montant total et le nombre total d’objets Il existe un test avant/après monnayeur sur le nombre d’objet total 1 point Il existe un test avant/après monnayeur sur le montant total 1 point

Menu de l'activité

  • 1 point code Créer inventaire doit remplir l'inventaire avec 9 produits avec des codes à barre valides
  • 1 point code Vider inventaire doit vider l'inventaire des produits existants
  • 1 point code Créer une liste doit créer une liste d'achats en prenant des produits au hasard avec des quantités aléatoires entre 1 et 5

Persistance

Produit

3 points test 1 point code Vous devez implanter la persistance d'un produit avec les champs appropriés. Votre code doit être validé par des tests automatisés.

Vous devez implanter les opérations suivantes au minimum (vous pouvez ajouter des opérations pour faciliter les tests ou autre).

Pour chaque méthode, vous devez tester les cas normaux et les cas exceptionnels (code barre absent etc.)

  • création d'un produit
  • récupération par son code barre
  • suppression de tous les produits

Achat

3 points test Vous devez vous assurer que votre calcul d'arrondi est correct dans tous les cas. Tous les arrondis sont testés .01 .02 .03 .04 etc .09

  • création d'un achat
  • récupération de l'ensemble des achats
  • autre cas ...

Fonctionnement global

5 points négatifs Votre application doit fonctionner sans plantage et correctement. Ce pointage fonctionne en négatif. Si l'application fonctionne correctement en tout temps, vous conservez votre note. Dans le cas contraire, vous perdez des points avec un maximum de 5.

  • plantage de l'application -1 point
  • interface illisible -1 point
  • information affichée incohérente -1 point
  • autre cas ...

Feuille de code barre

1 point négatif Vous devez joindre à votre remise une feuille avec la liste des 9 produits de votre inventaire avec une ligne par produit.

  • image du code barre
  • nom du produit
  • prix unitaire

Modèle ici

Vous devez également inclure une ligne vierge avec un code barre valide qui n'existe pas dans l'inventaire. L'absence de cette feuille entraîne la perte d'un point.
