Paiement

Vous devez implanter le nouveau processus de paiement incluant les étapes suivantes. Cela nécessitera la réutilisation de votre code du travail 2.

  • On appuie sur le bouton PAYER
  • 1 point code Un dialogue s'ouvre pour saisir les pièces et billets donnés par le client. On peut poursuivre uniquement si le montant est suffisant (voir à droite)
  • 1 point code Le système calcule le montant donné, le montant du, le montant à rendre en monnaie
  • 1 point code Le système affiche ces trois montants ainsi que les pièces billets qui sont rendus au client
  • Le ticket de caisse complet est affiché dans le log de l'appareil (avec le tag ACHATS):
    • 1 point code la liste des produits avec la quantité, un par ligne
    • 1 point code le montant hors taxe, le montant de taxes et le montant final
    • 1 point code la somme économisé grâce aux rabais appliqués

1 point code Une option est ajoutée au menu de l'activité pour visualiser l'état du tiroir caisse.

Taux de taxe

1 point code 1 point test Vous devez implanter 2 taux de taxes: un taux de taxe à 5% pour des produits essentiels comme le lait, le pain etc. Un autre taux de taxe à 35% pour les autres produits comme la pizza surgelé ou les sodas.

Vous devez inclure des produits pour chaque taux de taxe dans votre inventaire standard (vos 9 produits).

Rabais

Pour chaque type de rabais, vous devez :
  • programmer l'impact sur le ticket de caisse final
  • produire des tests unitaires pour valider le bon fonctionnement
  • ajouter les éléments d'interface graphique pour configurer le rabais

2 pour 1

1 point code 1 point interface graphique 1 point test Chaque produit peut être en 2 pour 1. Exemple : un client achète 7 bouteilles d’eau en 2 pour 1, on peut donc appliquer 3 fois le rabais: 

(1 + 1) + (1 + 1) + (1 + 1) + 1

Le client doit donc payer le prix de 4 bouteilles.

Pas de taxes si vous dépassez 100$

1 point code 1 point interface graphique 1 point test On doit pouvoir activer ce rabais qui annule les taxes quand le montant hors taxe dépasse 100$ pour un achat.

Combinaison de rabais

1 point code 2 points test Vous devez toujours appliquer les rabais de la manière la plus profitable au client.

Validation

4 points négatifs Tous les endroits où un utilisateur entre de l'information doivent être validés. Vous perdrez des points pour chaque endroit où j'arrive à entrer de l'information invalide. Ceci inclut le code barre ... 

public class BarcodeValidator {

    /**
     * Voir  http://en.wikipedia.org/wiki/Universal_Product_Code#Check_digits
     * @param s
     * @return
     */
    public static boolean isUPCValid(String s){
        if (s.length() != 12) return false;
        //Pattern.Pattern.compile("[0-9]*");
        if (!s.matches("[0-9]*")) return false;

        boolean odd = true;
        //Add the digits in the odd-numbered positions (first, third, fifth, etc.) together and multiply by three.
        int sumOdd = 0 ;
        //Add the digits in the even-numbered positions (second, fourth, sixth, etc.) to the result.
        int sumEven = 0;
        char[] array = s.toCharArray();
        for (int i = 0 ; i < array.length - 1 ; i++){
            int digit = Integer.parseInt(array[i]+"");
            //System.out.println("Digit " + digit);
            if (i%2==0) sumOdd += digit;
            else sumEven += digit;
            odd = !odd;
        }
        //Find the result modulo 10 (i.e. the remainder when divided by 10.. 10 goes into 58 5 times with 8 leftover).
        int result = (3*sumOdd + sumEven)%10;
        //If the result is not zero, subtract the result from ten.
        result = result!=0?10-result:0;
        // get the last digit value that should be equal to this result.
        int lastDigit = Integer.parseInt(array[array.length-1]+"");
        return result == lastDigit;
    }
}
