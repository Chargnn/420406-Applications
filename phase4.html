Rabais

Pour chaque type de rabais, vous devez :
  • programmer l'impact sur le ticket de caisse final (prix ou liste de produits)
  • produire des tests unitaires pour valider le bon fonctionnement
  • ajouter une option dans le menu de l'activité pour configurer le rabais

2 pour 1

Chaque produit peut être en 2 pour 1. Exemple : un client achète 7 bouteilles d’eau en 2 pour 1, on peut donc appliquer 3 fois le 2 pour 1 (1 + 1) + (1 + 1) + (1 + 1) + 1 ... le client doit donc payer le prix de 4 bouteilles.

Pas de taxes si vous dépassez 100$

On doit pouvoir activer ce rabais qui annule les taxes quand le montant hors taxe dépasse 100$ pour un achat.

Un produit gratuit par tranche de 60$

On doit pouvoir activer ce rabais qui ajoute un produit gratuit paramétrable pour chaque tranche de 60$.

Exemple : on veut donner un paquet de chips pour chaque tranche de 60$, un client qui achète pour 150$ dépasse 120$ il aura droit à 2 paquets.

Multilingue

L'intégralité de votre application doit être traduite en français et en anglais. La qualité de l'anglais n'est pas évaluée, vous pouvez utiliser Google Translate pour effectuer les traducations .

Validation

5 points négatifs Tous les endroits où un utilisateur entre de l'information doivent être validés. Vous perdrez des points pour chaque endroit où j'arrive à entrer de l'information invalide.
