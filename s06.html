Avant la séance (2h)

Vous devez lire la page suivante TDD

Ensuite vous devez commencer l'exercice d'intégration. Arrêtez vous quand vous avez complété vos 2 heures.

Pendant le cours à 40

Nous commencerons avec une explication de l' enum en Java.

Nous discuterons de quelques principes pour écrire vos tests.

  • Toujours pouvoir exécuter ce que j'écris, tests u fur et à mesure
  • Une classe de code = une classe de test
  • Une méthode = plusieurs tests
    • Quand ça fonctionne
    • Quand ça exceptionne
    • Quand ça boucle plante etc.
  • Comment utiliser la couverture de code

Nous discuterons également de plusieurs autres éléments concernant les tests

  • Quoi tester? Est-ce qu'on teste tout? Ce qui est risqué, nouveau, cher etc.
  • Quand doit-on tester? Quand est-ce qu'on écrit les tests? Quand est-ce qu'on les exécute?

Exercices de la semaine

Exercice NotesTests

Reprenez le code de l'exercice précédent Notes.

Pour chaque méthode de service, cherchez à identifier les cas problématiques pour les paramètres de la méthode.

Écrivez les tests pour valider le bon fonctionnement du code que vous aviez écrit. Apportez les correctifs si nécessaire. Validez le bon fonctionnement de votre application console existante.

Écrivez les tests qui
  • simulent une classe de 10 étudiants
  • assurez vous d'avoir un hasard reproductible
  • 3 épreuves avec des pointages au hasard entre 20 et 100 points
  • des notes aléatoires (entre 0 et le pointage de l'épreuve)
  • assurez vous que la moyenne est aux alentours de 50% (dû au tirage aléatoire)
  • afficher en console les valeurs tirées au hasard pour pouvoir les reporter dans Excel
  • valider les résultats de votre code avec une feuille de calcul Excel

Augmentez le nombre d'étudiants à 20000 et observez les résultats. Est-on plus proche de 50%?
